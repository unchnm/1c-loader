# 1C-loader — Загрузка изменённых файлов в базу 1С

Инструмент для разработчиков 1С. Определяет незакоммиченные изменения в XML/BSL-файлах
через `git diff` и загружает **только изменённые файлы** в выбранную базу 1С,
минуя полную загрузку конфигурации.

---

## Зачем это нужно

При разработке на 1С с хранением конфигурации в XML-формате (EDT/Конвертер конфигурации)
цикл разработки выглядит так:

```
Изменил BSL/XML → Загрузил в базу → Проверил
```

Стандартная «полная загрузка» занимает долго и грузит всё подряд.
**1C-loader** загружает только файлы, которые реально изменились — так намного быстрее.

---

## Требования

| Компонент | Версия |
|---|---|
| Windows | 10 / 11 (Vista+ для современного диалога папок) |
| PowerShell | 5.1 и выше |
| Git for Windows | любая актуальная |
| Платформа 1С:Предприятие | 8.3.x |

Никаких дополнительных зависимостей нет — скрипт вызывает `1cv8.exe` напрямую.

---

## Структура файлов

```
1C-loader.cmd                         ← двойной клик для запуска
scripts/
  Deploy-1C-Changes-GUI.ps1           ← GUI-приложение (основной файл)
  Deploy-1C-Changes.ps1               ← консольная версия (для скриптов/CI)
```

---

## Быстрый старт

1. Дважды кликнуть **`1C-loader.cmd`**
2. В поле **Папка проекта** нажать **Обзор** и выбрать корень git-репозитория
   (или папку, где лежит `Configuration.xml`)
3. Выбрать нужную **базу 1С** из выпадающего списка
4. Нажать **Прочитать изменения** — в списке появятся изменённые файлы
5. При необходимости снять галочки с ненужных файлов
6. Нажать **Загрузить выбранные (N)**

---

### Элементы управления

| Элемент | Описание |
|---|---|
| **Папка проекта** | Корень git-репозитория или папка с `Configuration.xml`. Можно ввести вручную или выбрать через диалог. |
| **Обзор...** | Открывает стандартный проводник Windows для выбора папки. |
| **База 1С** | Список баз из стандартного файла `ibases.v8i` (тот же список, что в запускаторе 1С). |
| **Прочитать изменения** | Запускает `git diff HEAD` и заполняет список изменённых XML/BSL-файлов. |
| **Список файлов** | Все файлы отмечены по умолчанию. Можно снять галочку с тех, которые не нужно загружать. |
| **Открыть конфигуратор** | Если отмечено — после успешной загрузки автоматически откроется Конфигуратор для выбранной базы. |
| **Загрузить выбранные (N)** | Запускает загрузку отмеченных файлов. Кнопка активна только при наличии отмеченных файлов. |

## Как это работает

1. **Поиск изменений** — `git diff HEAD --name-only --diff-filter=ACMR` возвращает
   список изменённых (в том числе проиндексированных) XML/BSL-файлов.

2. **Автоопределение корня конфигурации** — скрипт сам находит папку с `Configuration.xml`
   даже если репозиторий содержит и другие файлы (не только конфигурацию 1С).

3. **Прямой вызов платформы** — для каждой загрузки формируется командная строка и
   запускается `1cv8.exe DESIGNER` напрямую:
   ```
   1cv8.exe DESIGNER /S "server\base" /LoadConfigFromFiles "path\to\xml"
                     /files "CommonModules\Mod\Ext\Module.bsl,Documents\Doc\..."
                     /UpdateDBCfg
   ```

4. **Асинхронное выполнение** — UI не зависает во время работы платформы 1С
   (30–90 сек). Прогресс отображается счётчиком секунд в строке статуса.

## Аутентификация

По умолчанию используется **доменная аутентификация Windows** — `1cv8.exe` запускается
без параметров `/N` и `/P`, и платформа 1С подключается от имени текущего пользователя Windows.

Если нужна аутентификация по логину/паролю 1С — используйте консольную версию:

```powershell
.\scripts\Deploy-1C-Changes.ps1 -User "Администратор" -Password "пароль"
```

---

## Консольная версия

`Deploy-1C-Changes.ps1` — полнофункциональная консольная версия с интерактивным меню.
Поддерживает те же функции, плюс параметры командной строки:

```powershell
# Интерактивный режим (доменная аутентификация)
.\scripts\Deploy-1C-Changes.ps1

# Указать папку явно
.\scripts\Deploy-1C-Changes.ps1 -Path "D:\repositories\crm"

# Проверить список изменений без загрузки
.\scripts\Deploy-1C-Changes.ps1 -DryRun

# Аутентификация пользователем 1С
.\scripts\Deploy-1C-Changes.ps1 -User "Admin" -Password "secret"

# Не обновлять конфигурацию БД после загрузки
.\scripts\Deploy-1C-Changes.ps1 -SkipDbUpdate
```

## Структура git-репозитория

Скрипт ожидает стандартную структуру проекта 1С в XML-формате:

```
корень репозитория/
  .git/
  src/                         ← или cf/, config/, 1c/
    Configuration.xml          ← автоматически определяется
    CommonModules/
    Documents/
    Catalogs/
    ...
```

Если `Configuration.xml` лежит прямо в выбранной папке — это тоже работает.

---

## Список баз не отображается

Базы загружаются из файла `%APPDATA%\1C\1CEStart\ibases.v8i`.
Если список пуст:
1. Откройте стандартный запускатор **1С:Предприятие**
2. Добавьте нужные базы
3. Перезапустите 1C-loader
